🧭 Minecraft GeoGuessr – System Design Spec
🎯 Цел на проекта

Web multiplayer игра, вдъхновена от GeoGuessr, но върху Minecraft свят, вместо реална карта.

Играчите:

получават 360° панорама от случайна точка в Minecraft света
трябва да познаят координатите върху карта
получават точки според точността
играят в multiplayer (vs режим)
🧱 Общa архитектура
Frontend (React)
    ↕ Socket.IO
Backend (Node.js + Fastify)
    ↕ SQL
PostgreSQL (source of truth)

🎮 Game Loop (важно)
1. Lobby
players join match
backend създава match
2. Start game

Backend:

избира locations за rounds
изпраща:
round:start
3. Gameplay

Frontend:

показва panorama
player избира guess

Frontend → Backend:

socket.emit("guess", { x, z })
4. Backend обработка

Backend:

взима истинските координати
смята distance
смята score
връща резултат
socket.emit("round_result", {
  distance,
  score
});
5. End round
всички резултати се събират
запис в PostgreSQL
6. End match
total score се изчислява backend-side
запис в DB
leaderboard update
🧮 Scoring system

Backend изчислява:

distance = euclidean(x1, z1, x2, z2)

score = max(0, 5000 - distance * multiplier)

🧠 Critical rules
❌ НЕ:
frontend не смята score
frontend не смята distance
frontend не решава резултати
✅ ДА:
backend е единственият авторитет
DB пази финални резултати
socket е само transport
